GIT Y GITHUB


¿Que es Git?


Git es un sistema de control de versiones distribuido de código abierto. Su creador es Linus Torvalds, el cual lo creó en 2005. Se usa principalmente para el desarrollo de software.

Git permite a los desarrolladores mantener un historial compledo de los cambios en el código fuente de un proyecto, lo que facilita la colaboración y la gestion de versiones de un proyecto. El proyecto se puede dividir en ramas para que los desarrolladores puedan trabajar en estas y fusionar sus cambios de forma segura y eficiente.

Git se ejecuta en la linea de comandos, pero tambien tiene varias interfaces de usuario gráficas disponibles. Es muy usado por desarrolladores de software de todo el mundo, siendo una herramienta esencial en la gestión del desarrollo de software moderno.


Comandos Básicos de GIT:


GIT INIT
Este sirve para iniciar un repositorio en la carpeta principal de un proyecto.

GIT ADD
Añade los archivos que se mencionen al repositorio ( si se pone ".", se añadirá toda la carpeta en la que estes ).

GIT LOG
Muestra el historial de versiones.

GIT RESET
Descompone el archivo pero conserva el contenido del mismo.

GIT CLONE
Clona un repositorio existente.

GIT CONFIG
Accede a la configuración de GIT.

GIT STATUS
Enumera los archivos y su estado, si deben ser añadidos o comentados.

GIT DIFF
Muestra las diferencias de archivo que aún no se ponen en escena.

GIT --VERSION
Muestra la versión de GIT instalada.

GIT CONFIG --HELP
Muestra los comandos de GIT.
